Markdown 语法

Obsidian is a Markdown-based note-taking and knowledge base app.

We currently support the formats below:

Headers

This is a heading 2

This is a heading 3

This is a heading 4

This is a heading 5
This is a heading 6

Emphasis

This text will be italic
This will also be italic

This text will be bold
This will also be bold

You can combine them

Lists

  • Item 1
  • Item 2
    • Item 2a
    • Item 2b
  1. Item 1
  2. Item 2
  3. Item 3
    1. Item 3a
    2. Item 3b

http://obsidian.md - automatic!
Obsidian

Markdown style links can be used to refer to either external objects, such as web pages, or an internal page or image. If there are spaces in the url, they can be escaped by either using %20 as a space, such as Export options, or by enclosing the target in <>, such as [Slides Demo]().

Inline code

Text inside backticks on a line will be formatted like code.

Code blocks

Developer notes

We strive for maximum capability without breaking any existing formats, therefore we use a slightly unorthodox combination of flavors of markdown. It is broadly CommonMark, with the addition of some functionality from GitHub Flavored Markdown (GFM), some latex support, and our chosen embed syntax.
